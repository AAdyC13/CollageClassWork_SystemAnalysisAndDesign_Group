# PERT/CPM 圖
關鍵路徑： 設定題目 → 環境部署 → 架構設計 → 前後端開發 (最長路徑) → 整合測試 → 總測試修正 → 專題發表
``` mermaid
graph LR
    %% 階段 A: 規劃
    A1[設定題目與範圍] --> A2[調查產品功能]
    A2 --> A3[制定專題大綱]
    A3 --> A4[準備期初報告]
    A4 --> A5[研究開發工具]
    A5 --> A6[部署開發環境]

    %% 階段 B: 開發 (關鍵路徑開始)
    A6 --> B1[設計軟體架構圖]
    B1 --> B2[軟體前端開發]
    B1 --> B3[軟體後端開發]
    
    %% 並行開發與整合
    B2 --> B4[整合測試與優化]
    B3 --> B4
    B2 -.-> B5[模組插件開發]
    B5 --> B4

    %% 階段 C: 結案
    B4 --> C1[撰寫專題報告]
    B4 --> C2[輸出專案成品]
    C1 --> C3[總測試與修正]
    C2 --> C3
    C3 --> C4((專題發表))
```
